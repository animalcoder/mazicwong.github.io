框架

采用Scrapy

正文爬取

  1. 先在主函数中,从网易主页找出所有的URL,这些就是当天最热门的新闻,正则如下:
    url_pattern = r'(http://news\.163\.com)/(\d{2})/(\d{4})/(\d+)/(\w+)\.html'
  2. 然后对每个获取到的URL,直接获取他的source,date,newsId,cmtId,productKey,title,passage
  3. 有了productKey后,通过下面的构造方法,获得一个网易的API,在该API中可以获得一个boardId,这个boardId是用来构造评论地址的
    comments_api = 'http://comment.news.163.com/api/v1/products/' + productKey + '/threads/' + newsId
  4. 通过下面规则获取评论地址,comments
    comments = ('http://comment.news.163.com/'+boardId+'/'+newsId+'.html')
  5. 将上述获取到的数据放入item[]中,然后yield item

评论爬取

这里无法直接从新闻页面获取评论地址,要通过一些方法
地址格式: comments = ('http://comment.news.163.com/'+boardId+'/'+newsId+'.html')

  • 先从主页获取productKey和newsId
  • 然后构造出一个api地址,从该地址获取boardId
  • 通过boardId和newsId构造出评论地址
  • 到评论地址获取评论内容

坑点

记忆化搜索与DP区别

发表于 2017-10-04
字数统计: 976 | 阅读时长 ≈ 5

今天写DP专题的时候,看到一个题之前是做过的,当时用的记忆化搜索,这样重新用dp写了一遍,顺便总结下这两者的区别

hdu1078

题意很简单
n*n矩阵,老鼠吃奶酪,可以走四个方向,每次最多走k步
且要求每次找到的点的奶酪是递增的,问最多能吃到多少奶酪

法1:

明显,如果直接DP的话,虽然子问题重复,但是不满足无后效性,因为每次移动的方向不是固定的,而且四个方向,所以用记忆化搜索更好,直接搜索所有方向找到最优解

法2:

但是其实也可以DP,先将所有点按权值排列,这样找到一个方向后,可以进行DP

复杂度说明:

时间复杂度差不多,空间复杂度DP更好一点

总结:

  • DP:要有一个推进的方向,二维DP的话一般是从矩阵的左上角推到右下角,一维DP直接按照线性顺序推就可以了,
  • 这道题虽然是二维面,但是他的推进方向有四个,DP会有很多子问题,所以我们先按照权值排序,找到一个线性的关系,在这个关系上进行DP,就可以直接推进到整体最优解了

区间DP扩展:

区间DP的感觉,跟这里其实一样,在一个一维线性的序列中,我们没法按照一个固定的方向进行DP,所以我们按照区间长度来DP,对每个长度的区间,从左到右推一遍,然后扩大区间(详见hdu1078)

后缀数组

发表于 2017-09-14
字数统计: 4,170 | 阅读时长 ≈ 19

题型:

  1. 最长公共子串(poj2774):
    两个串连起来,然后找height[i]最大且sa[i]与sa[i-1]在两个串中
  2. 可重叠最长重复子串(poj3261):
    二分子串长度,然后用O(n)的check()判断height[]>k个数
  3. 前缀出现次数之和(hdu4552):
    转换为所有后缀与母串的lcp之和

总结:

  1. 后缀数组的精髓在于height[],做题的时候模板很简单.但是debug会很久,一般都是由于没有很好的考虑到各个变量或者数组的范围导致的
  2. MAXN的取值要考虑到的地方,一般取max(字符串长度len，字符最大值m，如果需要重复两次字符串先翻倍2len)

sa模板使用大全:

  1. 敲完倍增的模板,
  • 调用: build_sa(r,n,m),
  • 传入参数:
    • r是字符串str[]转换成int型字符后储存的数组；
    • n是字符串长度,直接传进去,不用n+1；
    • m要求大于字符串中最大的数,如果r[]是由字母转换来的,则m可以写28;
  • 返回: 得到Rank[i],sa[i],height[i]
    • Rank[i]: 第i个后缀的排名;
    • height[i]: 排名为i的后缀sa[i]与排名为(i-1)的后缀sa[i-1]的LCP(相邻排名最长公共前缀),height[]在很多人的模板里都是独立出来一个数组的,应该是参考了蓝书的做法,这里我给放进去build_sa[]了,不用的话在最后注释掉就好了
    • sa[i]: 排名为i的后缀位置;
  1. 敲完RMQ的模板，
  • 用处: 主要用来查询最长公共子串(lcp)，我们将lcp问题规约到RMQ问题，从而可以在O(n)内预处理，O(1)内查询出[a,b]范围内的lcp
  • 调用: RMQ::init(len+1)
  • 传入参数:
    • len+1是字符串长度+1
  • 返回: 得到init,rmq,lcp三个函数
    • init: 初始化RMQ各个参数
    • rmq: 用来查询的,只给lcp调用就好了
    • lcp[a,b]: 这个是大佬,可以O(1)查询任意两个串sa[a],sa[b]的最大公共子串
  1. dc3算法的模板,感觉没必要整理,直接用倍增的就好了

poj2774 后缀数组,最长公共子串,模板题

题意:
给你两串字符，要你找出在这两串字符中都出现过的最长子串

思路:
先用个分隔符将两个字符串连接起来，再用后缀数组求出height数组的值，找出一个height值最大并且i与i-1的sa值分别在两串字符中就好

hdu4552 后缀数组,求所有前缀出现的次数之和

思路:
等价于所有后缀与最长后缀的lcp之和,
因为后缀是和原串匹配的,所以可以理解为前缀
这样就可以把所有的前缀覆盖到!可以手写验证一下

这个题两种解法:

  • 法1是用rmq直接求出所有的lcp(0,i),然后加起来
  • 法2是从rank[0]开始,往左右两边扫,计算出最小的height[];
    法２其实道理很简单,只要理解了lcp的本质就可以了,lcp[a,b]是[a,b]两个串的最长公共子串,这个问题可以规约到rmq[height,a,b],而rmq[a,b]就是[a,b]段中最小的height[]值,所有一直扫描过去,答案是对的(这个方法好好理解!看了好几个小时终于懂了)

找出现至少k次的最长子串(可重叠)
