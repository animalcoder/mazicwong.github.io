Jnu寒假第五场

来源

poj3104

今天晚上的比赛遇到一个这样的题:
晾衣服：n件衣服各含a[i]水分，自然干一分钟一单位，放烘干机一分钟k单位，一次只能晒一件。求最短时间。

分析

一开始想用平均值做,但是把数据算一次平均值显然丢掉了一些与答案有关的精度,考虑能否用对k取模来存下精度,但是k到了1e9,所以思路又断了,最后这题还是比较明显用的二分答案思想

遇到了几个坑点

  • 题目的输出用’sample output #1’…其实这个是不用输出的…
  • 如果拿去烘干的话,每分钟是不会掉一滴水的,这个一开始没发现,英文太差的锅…
  • ans += ceil((double(a[i])-time)/k);是错的,这里每分钟掉k个水,
    按照k*x+(time-x)>=a[i]得到x>=(a[i]-time)/(k-1)
  • 还有一个很坑的点就是ans会爆int….这个也是忽略了…
  • 熟悉二分判答案的两种写法,看下面源码(具体还跟最大/小,取不取等号相关)

总之,这道题非常不错,值得回顾hh

netease爬虫

框架

采用Scrapy

正文爬取

  1. 先在主函数中,从网易主页找出所有的URL,这些就是当天最热门的新闻,正则如下:
    url_pattern = r'(http://news\.163\.com)/(\d{2})/(\d{4})/(\d+)/(\w+)\.html'
  2. 然后对每个获取到的URL,直接获取他的source,date,newsId,cmtId,productKey,title,passage
  3. 有了productKey后,通过下面的构造方法,获得一个网易的API,在该API中可以获得一个boardId,这个boardId是用来构造评论地址的
    comments_api = 'http://comment.news.163.com/api/v1/products/' + productKey + '/threads/' + newsId
  4. 通过下面规则获取评论地址,comments
    comments = ('http://comment.news.163.com/'+boardId+'/'+newsId+'.html')
  5. 将上述获取到的数据放入item[]中,然后yield item

评论爬取

这里无法直接从新闻页面获取评论地址,要通过一些方法
地址格式: comments = ('http://comment.news.163.com/'+boardId+'/'+newsId+'.html')

  • 先从主页获取productKey和newsId
  • 然后构造出一个api地址,从该地址获取boardId
  • 通过boardId和newsId构造出评论地址
  • 到评论地址获取评论内容

坑点

  • 函数没有用熟练,re.match(re.compile(pattern),str)是匹配pat串是否从函数开头开始的,要从中间找应该用findall

  • 调试时,用了bs4抓了一下所有的tag = script下来一个一个对比找问题..

  • http://lemon2013.top/2017/07/13/news-spider/
  • http://blog.reetsee.com/archives/237

sina

使用

scrapy startproject jnuxshc之后，要用到的就是进sina里面修改了
scrapy crawl mazic.py,spiders中爬虫代码,这里用main.py来执行了
最终接口,调用python3 main.py会得到一个jnu.csv的文件

需要修改的文件:

  • items.py: 修改为需要获得的数据
  • pipelines.py: 暂时不管
  • settings.py: 数据存储的地方和格式,修改robots,user_agent
  • middlewares.py: 暂时不管
  • spiders/*.py: 真正爬虫代码,可以用xpath,selector等处理,记得放入item中

Some Problem:

  1. 一开始运行完空白,看到debug中返回403,然后到settings.py里修改user_agent就好了
  2. 然后运行完还是爬不到,在settings把robots.txt修改为False就好了
  3. 第三个错误就是xpath写错的原因了,以后注意就行
  4. 由于输出到csv的列是无序的,所以在spiders/中加了csv_item_exporter.py,在settings.py中添加了FEED_EXPORTERS和FIELDS_TO_EXPORT

记忆化搜索与DP区别

今天写DP专题的时候,看到一个题之前是做过的,当时用的记忆化搜索,这样重新用dp写了一遍,顺便总结下这两者的区别

hdu1078

题意很简单
n*n矩阵,老鼠吃奶酪,可以走四个方向,每次最多走k步
且要求每次找到的点的奶酪是递增的,问最多能吃到多少奶酪

法1:

明显,如果直接DP的话,虽然子问题重复,但是不满足无后效性,因为每次移动的方向不是固定的,而且四个方向,所以用记忆化搜索更好,直接搜索所有方向找到最优解

法2:

但是其实也可以DP,先将所有点按权值排列,这样找到一个方向后,可以进行DP

复杂度说明:

时间复杂度差不多,空间复杂度DP更好一点

总结:

  • DP:要有一个推进的方向,二维DP的话一般是从矩阵的左上角推到右下角,一维DP直接按照线性顺序推就可以了,
  • 这道题虽然是二维面,但是他的推进方向有四个,DP会有很多子问题,所以我们先按照权值排序,找到一个线性的关系,在这个关系上进行DP,就可以直接推进到整体最优解了

区间DP扩展:

区间DP的感觉,跟这里其实一样,在一个一维线性的序列中,我们没法按照一个固定的方向进行DP,所以我们按照区间长度来DP,对每个长度的区间,从左到右推一遍,然后扩大区间(详见hdu1078)
