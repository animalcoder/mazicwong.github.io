G++&C++

最近写了几道字典树,发现Trie用动态内存分配实现的话,交到OJ上用G++会MLE,用C++不会,
原因: 群里大佬说G++指针的size是C++的两倍多,会MLE,然后不断的new出来的内存也是不连续的,跑起来会变慢,测了一下时间大概变慢了一半

顺便整理下这几个的注意点吧

手动扩大栈内存

一般,用g++提交时间慢,用C++提交会栈溢出

g++: 用读入挂防止TLE
c++: #pragma comment(linker, “/STACK:102400000,102400000”)
